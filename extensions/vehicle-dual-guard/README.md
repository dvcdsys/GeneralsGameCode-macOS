# Vehicle Dual Guard — Guard-From-Position add-on

Loose-INI add-on that puts the **Guard-From-Position** ("double guard") command
on every **Cold War Crisis** combat **vehicle** and **aircraft** — and on
nothing else (no infantry, no unarmed/support vehicles, no civilian props).

> **Easiest install:** the **GeneralsZH Launcher → Extensions** tab installs and
> removes this with one click (it copies these files into your game-data folder).
> The manual `install.sh` / copy steps below still work if you prefer the CLI.

The Guard-From-Position *mechanic* itself lives in the macOS-port engine (this
project). If you maintain CWC and want different attribution, please open an issue.

Guard-From-Position is a two-click order:

1. first click sets the unit's **home** position;
2. second click sets a **watch zone** — the unit attacks anything that enters
   it and returns home when the threat clears.

## What's in here

```
Data/INI/OverrideCommandButton/_zz_GuardFromPosition.ini   ← the command button
Data/INI/OverrideCommandSet/_zz_GuardFromPosition.ini      ← 56 patched CommandSets
```

These are **purely additive loose overrides**. They do **not** modify CWC's
`_469_CWC.gib` archive in any way — the engine loads them through
`INI_LOAD_CREATE_OVERRIDES` and chains them on top of the mod's base
CommandSets at runtime. Removing the two files fully reverts the add-on.

> Requires the macOS-port engine changes that make the engine re-load loose
> `OverrideCommandButton/` + `OverrideCommandSet/` dirs across map transitions
> and re-cache their button images (commit "feat(zerohour): Guard-from-Position
> mechanic + CWC loose-override mechanism"). On a stock Windows build the
> `GUARD_FROM_POSITION` command + button image must also exist.

## Install

Copy the `Data/` tree into your CWC-enabled Zero Hour install root (the folder
that already contains `_469_CWC.big` / the CWC `.gib`), merging with any
existing `Data/INI/Override*` dirs:

```sh
./install.sh "/path/to/Command and Conquer Generals Zero Hour/Command and Conquer Generals Zero Hour"
```

or manually drop the two `.ini` files into
`…/Data/INI/OverrideCommandButton/` and `…/Data/INI/OverrideCommandSet/`.

To uninstall, delete the two `_zz_GuardFromPosition.ini` files.

## Coverage

Generated by [`scripts/cwc_guard_patch.py`](../../scripts/cwc_guard_patch.py)
against CWC 1.5. It classifies by each Object's `KindOf` + weapons:

* **Included** — `KindOf` has `VEHICLE` or `AIRCRAFT` **and** the unit can shoot
  (`CAN_ATTACK`, or a real `PRIMARY` weapon). Covers all tanks, IFVs/APCs,
  artillery, SAM/AA, attack/transport helicopters and jets, including
  `CommandSetUpgrade`/veterancy command-set variants.
* **Excluded** — all `INFANTRY`; unarmed/support vehicles (no `CAN_ATTACK`);
  the shared `GenericCommandSet` / `StopOnlyGenericCommandSet` fallbacks
  (patching them would leak the button onto non-combat objects); `Civilian*`
  mission props.

On CWC 1.5 that selects **203 / 1747 Objects → 66 CommandSets patched (incl. rider weapon-swap variants)**. The
button is added at each set's lowest free visible slot (1–14) without
overwriting any existing command. Three full transports
(`BTR80`, `BTR80p`, `Mi8`) have no free visible slot and are skipped.

## Regenerate

```sh
python3 scripts/cwc_guard_patch.py \
  "/path/to/.../GLM/Cold War Crisis/1.5/_469_CWC.gib" \
  "/path/to/Command and Conquer Generals Zero Hour/Command and Conquer Generals Zero Hour"
```

This rewrites the two files in the install's `Data/INI/Override*` dirs; copy
them back here to refresh the packaged copy.
